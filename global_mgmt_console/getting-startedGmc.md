# Accessing the IBM Cloud Backup and Recovery Manager from your instance of Backup and Recovery
{: #getting-started-backup-recovery-gmc}

{{site.data.keyword.baas_full}} provides complete backup solutions for workloads across IBM Cloud. It introduces a Backup and Recovery Manager for centralized monitoring, reporting, and alerts, enabling customers to manage all backup and recovery instances through a single, intuitive user interface.
{: shortdesc}



## Before you begin
{: #baas-getting-started}

You need the following to get started with IBM Cloud Backup and Recovery Manager:

- An [{{site.data.keyword.cloud}} Platform account](https://cloud.ibm.com)
- An instance of {{site.data.keyword.baas_full_notm}} 

If you do not have an instance of {{site.data.keyword.baas_full_notm}}, follow the steps that are listed in [Getting started with IBM CloudÂ® Backup and Recovery](/docs/backup-recovery?topic=backup-recovery-getting-started-backup-recovery).


## Overview
{: #baas-overview}

The Backup and Recovery Manager offers a unified interface for managing all Backup and Recovery service instances. It provides centralized monitoring of {{site.data.keyword.baas_full_notm}} instances in a region under a single IBM account. You can see key details such as protection coverage, recovery and protection status by type, and alerts. It also shows all triggered alerts and allows you to filter, view, and download reports.





## Accessing Backup and Recovery Manager
{: #baas-provision-console}

To access the Backup and Recovery Manager, you need to have an active {{site.data.keyword.baas_full_notm}} instance. If you already have an active instance, select it and skip steps 1 to 6.

1. Log in to your IBM cloud [account](https://cloud.ibm.com/){: external}.
1. Go to the catalog by clicking **Catalog** from the navigation bar.
1. Search the catalog for **Backup and Recovery**.
1. Give the service instance a name.
2. Select a region.
3. Click **Create** after accepting the license agreements.
4. Open your {{site.data.keyword.baas_full_notm}} instance.
4. In the left navigation window, click **Backup Manager**, and select the region that you want to see the report for the Backup and Recovery Manager.
5. The Backup and Recovery Manager dashboard displays key details such as Protection Coverage, Recovery Status by Type, Protection Status by Type, and Alerts for all instances within the selected region.
