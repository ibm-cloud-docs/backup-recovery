# Protection group Run Now
{: #data-source-connector-iks-roks}

The Run Now feature allows you to immediately start a backup for the objects included in a Protection Group, without waiting for a scheduled policy.
This is useful for:
- Testing backups
- Taking an on‑demand snapshot before changes
- Re-running backups after a failure
- Capturing incremental changes quickly

When you click `Run Now`, the system presents different backup options based on the protection group configuration and the state of the last backup.
You can manually start a protection run of the Object that you have created to back up the namespaces of the Kubernetes cluster. It provides you with options to:
- Back up all the Objects or only selected Objects.
- Select an incremental or full backup.

To start a protection run:
1. In the BRS instance Dashboard, navigate to `Data Protection` \> `Protection`.
2. On the Protection Source page, select the protection group you want to run the backup for.
3. Click on the three dots and click `Run Now`.
4. In the pop-up window click pencil icon for edit, perform the following:
- In the `Backup` drop-down list, select any one of the following:
   - Backup all Objects
   - Backup only selected Objects
- In the `Backup Type` drop-down list, select the backup type. By default, an archival backup will be performed.
5. Click `Run Now`.

## Pause future runs
{: #iks-roks-pause-run}

You can pause future runs for the Objects. However, if the Object is currently executing a protection run, the current run continues to execute and only future runs of the Object are paused.
To pause future run:
1. In BRS instance Dashboard,  navigate to `Data Protection` \> `Protection`.
2. On the Protection group page, select the protection group you want to pause backup for.
3. Click the three dots click `Pause Future Runs`.

## Resume paused protection runs
{: #iks-roks-resume-run}

You can resume a paused protection run. When you resume a protection run, if the backup snapshots are expired then a full backup is triggered.
To resume a paused protection run:
1. In BRS instance Dashboard,  navigate to `Data Protection` \> `Protection`.
2. On the Protection group page, select the protection group you want to resume backup for.
3. Click `Resume`.

## Delete the object
{: #iks-roks-delete object}

You can delete an Object if you:
- Do not want to retain the backups and the local and archived snapshots that are generated by the Object.
- Want to reclaim the storage space utilised by the Object.

You can delete an Object in the following ways:
- Delete Object Only: Allows you to delete the Object. All the backups generated by the Object are preserved till the snapshots expire.
- Delete Object and Snapshots: Allows you to delete the Object and all the local and archived Snapshots generated by the Object.

After deleting the Object and Snapshots, the reclaimed storage space details are not displayed immediately. It will take some time to reflect on the BRS Dashboard.

To delete the Object:
1. In BRS instance Dashboard, navigate to Data Protection> Protection
2. On the Protection group page, select the protection group you want to delete backup for.
3. Click three dots and delete (the protection group will get deleted only if the backup is paused).
4. In the **Delete Protection Group?** dialog box, select any one of the following options:

   - Delete Object Only
   - Delete Object and Snapshots

## Backup types available under Run Now
{: #iks-roks-backup-types-run-now}

When triggering a manual backup, you may see the following `Backup Types` available under `Run Now`:

| Section | Details |
|--------|---------|
| **Full Backup** | Captures all selected data regardless of previous backup history. <br> Can run for: all objects, selected objects, or objects that succeeded in the last run. <br> Suggested use: complete data copy, first backup of new protection group, and reset incremental chain. |
| **Incremental Backup** | Captures only changed data since the last successful backup. <br> Can run for: all objects, selected objects, or succeeded objects (only if previous run had partial or full success). <br> Suggested use: quick backups and protect recent changes. |

Here are the details of the each selection option:

| Selection Option | Description | Scenarios / Notes |
|------------------|-------------|-------------------|
| **All Objects in the Protection Group (full or incremental)** | Runs a backup (full or incremental) on every object defined in the group. Use when you want consistent coverage of the entire environment. | Always available. |
| **Selected Objects in the Protection Group (full or incremental)** | Allows manual selection of objects to back up. Useful when backing up only a specific namespace or testing backups for a particular object. | Always available. |
| **All Objects that succeeded in the Last Run (full or incremental)** | Appears only if the previous backup completed successfully. Uses the success list from the last run and backs up only those objects. Ensures consistency and avoids running backups on unknown or incomplete object sets. | **Appears when:** Last run succeeded; Partial success (for example, 2 out of 5 namespaces succeeded). <br> **Hidden when:** Previous run cancelled; Previous run failed completely; No successful run exists. |


**Backup option visibility matrix**

| Scenario | All objects that succeeded last run | All objects in the protection group | All objects that succeeded in the last run |
|----------|------------------------------------|--------------------------------------|----------------------------------------------|
| Last run succeeded | appears | appears | appears |
| Last run failed or was cancelled | hidden | appears | appears |
| No previous backup exists | hidden | appears | appears |
